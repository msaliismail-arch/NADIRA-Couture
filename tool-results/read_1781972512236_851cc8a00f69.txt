    27→- Created shared auth helper `src/lib/admin-auth.ts`:
    28→  * `createToken(admin)`: HMAC-SHA256-signed token = base64url(JSON payload) + "." + base64url(hmac), using ADMIN_SECRET env (default `nadira-couture-secret-2024`). Payload includes id/email/role/ts.
    29→  * `verifyToken(token)`: re-signs and constant-time-compares; returns decoded `{id,email,role}` or null.
    30→  * `requireAdmin(request)`: extracts `Authorization: Bearer <token>`, verifies, fetches Administrateur row from DB, returns null on any failure.
    31→  * `verifyPassword`/`hashPassword` wrappers around bcryptjs.
    32→- Created reference helper `src/lib/reference.ts` with `generateReference()` → `NAD-YYYYMMDD-XXXX` (4 random digits) + `uniqueReference()` collision-safe variant.
    33→- Built admin auth routes:
    34→  * `POST /api/admin/login` — verifies email+password against Administrateur table with bcrypt.compare, returns `{token, admin:{id,nom,email,role}}` or 401.
    35→  * `GET /api/admin/me` — `requireAdmin`, returns current admin info or 401.
    36→- Built content + categories routes:
    37→  * `GET/PUT /api/contenu` — GET returns all rows or one via `?cle=`; PUT upserts `{cle,valeur}` (admin).
    38→  * `GET /api/categories` — list ordered by ordre with `_count` of produits.
    39→- Built products routes:
    40→  * `GET /api/produits` — supports `?categorie=slug`, `?vedette=true`, `?recherche=xxx` (OR on nom/description/tissu), `?couleur=xxx` (contains), `?tissu=xxx` (contains); combines filters; includes categorie.
    41→  * `POST /api/produits` — admin create with full body validation.
    42→  * `GET /api/produits/[slug]` — slug OR numeric id; includes categorie + artisans (via ProduitArtisan→Artisan).
    43→  * `PUT /api/produits/[slug]` — admin update (partial), syncs `artisanIds` if provided.
    44→  * `DELETE /api/produits/[slug]` — admin delete.
    45→- Built artisans routes:
    46→  * `GET /api/artisans` — ordered by ordre, includes produits (select id/nom/slug).
    47→  * `POST /api/artisans` — admin create with optional `produitIds` link.
    48→  * `PUT /api/artisans/[id]` — admin update + optional produitIds resync.
    49→  * `DELETE /api/artisans/[id]` — admin delete.
    50→- Built commandes routes:
    51→  * `GET /api/commandes` — admin list (with client + lignes.produit) ordered by dateCommande desc; `?reference=xxx` (public) returns single commande with lignes.produit.categorie for tracking.
    52→  * `POST /api/commandes` (public) — find-or-create Client by telephone, optional Mesure creation, validates produits, computes montantTotal from produit.prix, generates `NAD-YYYYMMDD-XXXX` reference, statut="en_attente", returns `{commande, reference}`.
    53→  * `GET/PUT/DELETE /api/commandes/[id]` — admin only; PUT updates statut/acompte/modePaiement/notes/dateRetrait/montantTotal; DELETE removes lignes then commande.
    54→  * `POST /api/commandes/manuelle` — admin manual order with `idClient` OR `newClient` body, optional `mesures`, per-line `prixUnitaire` override, custom statut/modePaiement/acompte.
    55→- Built mesures routes:
    56→  * `GET /api/mesures` — admin list; `?telephone=xxx` (public) returns that client's mesures (for sur-mesure prefill).
    57→  * `POST /api/mesures` — public create via `idClient` OR `telephone` (resolves existing client).
    58→- Built rendezvous routes:
    59→  * `GET /api/rendezvous` — admin list with client include.
    60→  * `POST /api/rendezvous` (public) — creates RDV with statut="planifie", auto-links to Client by telephone if found.
    61→  * `PUT/DELETE /api/rendezvous/[id]` — admin only; PUT updates statut/notes.
    62→- Built avis routes:
    63→  * `GET /api/avis` — `?approuve=true` (public) → approved only; admin token returns ALL with commande.client include; defaults to approved-only for anonymous callers.
    64→  * `POST /api/avis` (public) — creates with approuve=false, validates note 1-5.
    65→  * `PUT /api/avis` — admin approve/unapprove via `{id, approuve}`.
    66→  * `DELETE /api/avis/[id]` — admin delete.
    67→- Built clients routes:
    68→  * `GET /api/clients` — admin list with `_count` of commandes.
    69→  * `POST /api/clients` — public find-or-create by telephone (fills missing email/adresse on existing).
    70→  * `GET /api/clients/[id]` — admin single with commandes.lignes.produit + mesures + rendezVous.
    71→- Built stats route `GET /api/stats` (admin) — parallel Prisma queries:
    72→  * totalCommandes, commandesEnAttente/EnConfection/Livrees counts.
    73→  * caTotal (sum livree.montantTotal), caMois (sum livree this month).
    74→  * totalClients, totalProduits, rdvAVenir (statut in [planifie,confirme] && dateRdv >= now).
    75→  * topProduits via `ligneCommande.groupBy` by idProduit (_sum quantite desc, take 5) joined with produit records.
